### 决策

- **资源不放 git-lfs**：97 MB 都在 github.com 100 MB/file + 1 GB/repo 限制内；最大单文件 3 MB；直接 commit 简单可靠
- **mesh-type geom 暂用小球占位**：121 个机器人 mesh geom，如果都渲成 wireframe bbox 视觉太吵；3cm 彩色小球作为"骨架"够用，真 OBJ loader 放 Phase 1b
- **不用 Next.js / Tailwind**：纯静态托管 Vite 够；Tailwind Phase 7 UI 外壳再议
- **L1 不用真 LLM**：浏览器版走 35 intent 关键词映射，零 fetch 零 API key（已在项目约束里写死）

### 踩坑记录（带入 docs/tech-notes.md）

1. **mujoco-js 0.0.7 只收新 MuJoCo 3.3 API**：geom 上的 `shellinertia="true"` 被拒；必须用 mesh 上的 `inertia="shell"`
2. **exception 解码**：`mujoco.MjModel.loadFromXML` 抛 emscripten 包装对象，`e.message` 是 undefined；必须 `mujoco.getExceptionMessage(e?.excPtr ?? e)` 返回 `[name, message]`
3. **cam_mode 枚举**：0=FIXED, 1=TRACK, 2=TRACKCOM, **3=TARGETBODY**, 4=TARGETBODYCOM（第一版错成 1/2）
4. **WASM 以 base64 内嵌**：mujoco_wasm.js 11MB 因为嵌了整个 WASM；无需单独 fetch `.wasm`，无需 Vite wasm 插件
5. **Stretch 自带机载相机**：`<include>` 引入的机器人 5 个 onboard cam 会叠加进 `model.ncam`，hospital_ward_min 最终 ncam=10 不是 5
6. **Z-up vs Y-up**：MuJoCo Z-up，Three 默认 Y-up；修相机 `.up.set(0,0,1)`，世界坐标不动
7. **pip 不在 uv venv 里**：`demo/core/.venv` 是 uv 管的，没 pip；`python -m ensurepip` 后 pip install 又失败（stretch_mujoco 不在 PyPI）→ 直接 git clone 上游仓库
